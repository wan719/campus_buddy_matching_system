# 可行性分析报告

## 一、技术可行性

### 1.1 技术栈评估

| 技术 | 熟悉程度 | 可行性说明 |
|------|----------|------------|
| Spring Boot | ✅ 熟悉 | 主流框架，生态完善 |
| MySQL | ✅ 熟悉 | 关系型数据库，稳定可靠 |
| MyBatis | ✅ 熟悉 | 灵活控制SQL |
| Spring Security + JWT | ✅ 熟悉 | 认证授权方案成熟 |
| Redis | ⚠️ 学习中 | 用于缓存和会话管理，学习成本低 |
| Gradle | ⚠️ 首次使用 | 与Maven类似，迁移成本可控 |
| Vue | ⚠️ 首次使用 | 之前用HTML/CSS/JS，Vue学习曲线平缓 |

### 1.2 开发工具

| 工具 | 用途 | 可行性 |
|------|------|--------|
| VS Code | 主开发工具 | ✅ 免费，插件丰富 |
| IntelliJ IDEA | 辅助开发 | ✅ 学生免费版可用 |
| DeepSeek / ChatGPT / Codex | AI辅助编码 | ✅ 可提高开发效率 |

### 1.3 技术风险评估

| 风险 | 应对措施 |
|------|----------|
| Gradle 构建问题 | 提前学习基础语法，参考官方文档 |
| Vue 学习成本 | 使用 Vue CLI 脚手架，先做简单页面 |
| Redis 集成 | 先从本地环境开始，逐步引入 |

**结论**：技术可行性 ✅ **可行**

## 二、经济可行性

### 2.1 成本分析

| 项目 | 费用 | 说明 |
|------|------|------|
| 开发工具 | ¥0 | VS Code 免费，IDEA 学生免费 |
| 代码托管 | ¥0 | GitHub 免费仓库 |
| ChatGPT Plus | ¥140/月 | 可选，用于提高效率 |
| 云服务器 | ¥100-300/年 | 学生优惠，如阿里云/腾讯云 |
| 域名 | ¥0-50/年 | 可选，也可用 IP 访问 |
| **总计** | **¥100-500/年** | 主要为服务器费用 |

### 2.2 成本优化建议
- 开发阶段使用本地环境，不购买服务器
- ChatGPT 可使用免费版，或国产平替（DeepSeek、通义灵码等）
- 部署阶段再购买服务器，按量付费

**结论**：经济可行性 ✅ **可行**（个人可承受）

## 三、运营可行性

### 3.1 初期运营
- **用户量**：前期预计 10-50 人试用
- **内容审核**：单人可管理（举报机制 + 人工审核）
- **用户增长**：通过朋友圈、校园论坛推广

### 3.2 长期运营挑战

| 挑战 | 应对思路 |
|------|----------|
| 用户留存 | 增加信用分、评价体系，提高用户粘性 |
| 内容审核 | 后续可增加敏感词过滤、管理员后台 |
| 恶意行为 | 举报机制 + 信用分惩罚 |

**结论**：运营可行性 ✅ **可行**（前期可控）
