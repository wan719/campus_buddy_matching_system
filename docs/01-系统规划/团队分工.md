# 团队分工

## 角色与职责

| 角色 | 负责人 | 职责 |
|------|--------|------|
| 产品经理 | 本人 | 需求分析、用户故事、验收标准 |
| 后端开发 | 本人 | 架构设计、API开发、数据库设计 |
| 前端开发 | 本人 | 页面实现、前后端联调 |
| 测试工程师 | 本人 | 单元测试、集成测试、压力测试 |
| 运维工程师 | 本人 | 服务器配置、部署上线、监控维护 |
| 文档编写 | 本人 | 需求文档、设计文档、用户手册 |

## 开发模式
采用 **单人敏捷开发** 模式：
- 每周末进行 Sprint 回顾
- 每日任务清单管理
- 使用 GitHub Projects 看板跟踪进度

## 协作工具
| 用途 | 工具 |
|------|------|
| 代码管理 | GitHub |
| 任务跟踪 | GitHub Projects / 飞书任务 |
| 文档管理 | GitHub + Markdown |
| 沟通记录 | 本 Markdown 文档 |
