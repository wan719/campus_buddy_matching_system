# 技术选型说明

## 一、后端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| **Spring Boot** | 4.04 | 主流 Java 框架，生态完善，开发效率高 |
| **Spring Security** | 6.x | 安全框架，与 JWT 无缝集成 |
| **JWT** | - | 无状态认证，适合前后端分离 |
| **MyBatis** | 3.x | 灵活编写 SQL，支持复杂查询 |
| **MySQL** | 8.0 | 稳定可靠，支持 JSON 字段 |
| **Redis** | 7.x | 缓存、分布式会话（后期引入） |
| **Gradle** | 8.x | 构建工具，首次尝试，学习新技能 |

### 为什么选择这些技术？
- **Spring Boot**：快速搭建项目，自动配置，减少样板代码
- **MyBatis**：相比 JPA 更灵活，适合需要优化 SQL 的场景
- **JWT**：实现无状态认证，方便扩展

## 二、前端技术栈

| 技术 | 选型理由 |
|------|----------|
| **Vue 3** | 首次学习，组件化开发，生态成熟 |
| **HTML/CSS/JavaScript** | 基础保障，作为兜底方案 |
| **Element Plus** | Vue 3 组件库，快速搭建后台管理界面 |

### 前端学习计划
1. 先用 HTML/CSS/JS 实现核心页面原型
2. 边开发边学习 Vue 3，逐步重构
3. 最终完成前后端分离

## 五、技术选型总结

| 层级 | 选择 | 熟悉程度 |
|------|------|----------|
| 后端框架 | Spring Boot | ✅ 熟悉 |
| 安全认证 | Spring Security + JWT | ✅ 熟悉 |
| 数据库 | MySQL + MyBatis | ✅ 熟悉 |
| 缓存 | Redis | ⚠️ 学习中 |
| 构建工具 | Gradle | ⚠️ 首次使用 |
| 前端 | Vue 3 | ⚠️ 首次使用 |
| AI辅助 | DeepSeek / ChatGPT | ✅ 熟练 |

**选型原则**：优先使用熟悉技术，适度引入新技术作为学习机会。
