# 项目背景与目标

## 一、项目背景

### 1.1 现实问题
当代大学生正处于从校园走向社会的“雏鹰阶段”，社交圈相对封闭，寻找志同道合的伙伴存在以下痛点：

| 痛点 | 描述 |
|------|------|
| **渠道缺失** | 缺乏专门的大学生搭子匹配平台 |
| **QQ频道信息过载** | 消息刷新快，重要内容被淹没 |
| **无法精准筛选** | 无法按类型（学习/运动）、时间、地点分类搜索 |
| **信任成本高** | 陌生人社交存在安全顾虑，缺乏信用体系 |

### 1.2 项目定位
**校园搭子匹配系统 2.0** 是一个专为在校大学生设计的社交匹配平台，帮助用户快速找到学习、运动、吃饭等各类“搭子”，解决大学生“找人难、匹配难、信任难”的问题。

---

## 二、项目目标

### 2.1 核心目标
> 搭建一个**安全、高效、真实**的校园搭子匹配系统，让每一位大学生都能轻松找到志同道合的伙伴。

### 2.2 具体目标

| 目标类型 | 具体内容 |
|----------|----------|
| **功能目标** | 实现用户注册/登录、搭子活动发布、智能匹配、申请审核、信用评价等核心功能 |
| **技术目标** | 采用 Spring Boot + MyBatis + MySQL 主流技术栈，代码规范，文档齐全 |
| **实践目标** | 完整走通“需求分析 → 系统设计 → 编码实现 → 部署上线”全流程，产出工业级项目 |
| **交付目标** | 产出可运行的线上系统 + 完整的项目文档（需求规格说明书、设计文档、部署手册） |

### 2.3 成功标准
- [ ] 用户可以正常注册/登录，身份通过学号验证
- [ ] 用户可以发布搭子活动，并设置类型、时间、地点
- [ ] 用户可以按条件搜索活动并申请加入
- [ ] 活动发起人可审核申请者
- [ ] 活动结束后可互相评价，信用分动态更新
- [ ] 系统可稳定部署在云服务器上对外访问
