## 第二部分：模块化与原生开发方式（第2学时）

### 2.1 为什么需要模块化？

#### JavaScript模块化的历史

在ES6之前，JavaScript没有内置的模块系统，这导致了严重的命名空间污染问题：

```javascript
// ❌ 问题：全局命名空间污染
// file1.js
var name = '张三';
var user = { name: name, age: 25 };

// file2.js
var name = '李四';  // 覆盖了 file1.js 的 name！
var user = { name: name, age: 30 };  // 冲突！

// index.html
<script src="file1.js"></script>
<script src="file2.js"></script>
// 结果：name 和 user 被覆盖，程序出错！
```

**模块化的价值：**

```
✅ 避免全局命名空间污染
✅ 提高代码可维护性
✅ 增强代码复用性
✅ 明确代码依赖关系
✅ 支持按需加载
```

#### 模块化规范的演进

```
历史演进：
─────────────────────────────────────────────────
2009年  →  Node.js诞生（使用CommonJS）
2013年  →  AMD规范（RequireJS）
2014年  →  CMD规范（SeaJS）
2015年  →  ES6官方推出ES Modules（import/export）
2020年代 → ES Modules成为标准，浏览器全面支持
─────────────────────────────────────────────────
```

#### ES6+ 箭头函数（Arrow Functions）

在深入学习模块化之前，我们还需要了解 **箭头函数** 语法。这是ES6（ES2015）引入的重要特性，让函数写法更简洁。

**为什么需要箭头函数？**

```javascript
// ❌ 传统函数写法（比较啰嗦）
function add(a, b) {
  return a + b;
}

const multiply = function(a, b) {
  return a * b;
};

// ✅ 箭头函数写法（简洁明了）
const add = (a, b) => a + b;
const multiply = (a, b) => a * b;
```

**基本语法：**

```javascript
// 1. 无参数的箭头函数
const sayHello = () => {
  console.log('Hello!');
};

// 2. 单个参数（可以省略括号）
const double = n => n * 2;
// 等价于：
// const double = (n) => { return n * 2; };

// 3. 多个参数（需要括号）
const add = (a, b) => a + b;

// 4. 函数体有多条语句（需要花括号和 return）
const calculate = (a, b) => {
  const sum = a + b;
  const product = a * b;
  return sum + product;
};

// 5. 返回对象字面量（需要用括号包裹）
const createUser = name => ({ name: name, id: Date.now() });
// 注意：如果不加括号，{ } 会被误认为是函数体
```


**在回调函数中的应用（箭头函数最常用的场景）：**

```javascript
// 示例1：数组方法
const numbers = [1, 2, 3, 4, 5];

// ❌ 传统写法
const doubled = numbers.map(function(n) {
  return n * 2;
});

// ✅ 箭头函数（更简洁）
const doubled = numbers.map(n => n * 2);

// 示例2：filter
const evens = numbers.filter(n => n % 2 === 0);

// 示例3：reduce
// array.reduce(callback(accumulator, currentValue), initialValue)
const sum = numbers.reduce((acc, n) => acc + n, 0);
```

**箭头函数与 this：**

```javascript
// ⚠️ 注意：箭头函数没有自己的 this
const obj = {
  name: 'Alice',

  // 传统函数：this 指向调用者
  greetTraditional: function() {
    setTimeout(function() {
      console.log(this.name);  // undefined（this 指向全局对象）
    }, 100);
  },

  // 箭头函数：this 继承外层作用域
  greetArrow: function() {
    setTimeout(() => {
      console.log(this.name);  // 'Alice'（this 继承自 greetArrow）
    }, 100);
  }
};
```

**在课程示例中的应用：**

接下来的模块化代码中，你会看到大量箭头函数的使用：

```javascript
// 工具函数（简洁明了）
export const add = (a, b) => a + b;
export const capitalize = (str) => str.charAt(0).toUpperCase() + str.slice(1);

// 数组方法的回调函数
export async getUserById(id) {
  const users = await getUsers();
  return users.find(u => u.id === id);  // 箭头函数作为回调
}

// 事件处理（常见于前端开发）
document.getElementById('btn').addEventListener('click', () => {
  console.log('按钮被点击');
});
```

**重要提示：**

```javascript
// ❌ 错误：箭头函数不能用作构造函数
const Person = (name) => {
  this.name = name;
};
const p = new Person('Alice');  // TypeError: Person is not a constructor

// ✅ 正确：需要构造函数时使用传统函数
function Person(name) {
  this.name = name;
}
const p = new Person('Alice');  // 正确

// ❌ 错误：箭头函数没有 arguments 对象
const func = () => {
  console.log(arguments);  // ReferenceError: arguments is not defined
};

// ✅ 正确：使用剩余参数代替
const func = (...args) => {
  console.log(args);  // 正确
};
```

**学习建议：**

1. **优先使用箭头函数**：对于简单的函数、回调函数，箭头函数是首选
2. **注意 this 绑定**：如果需要访问 this，考虑箭头函数的行为
3. **构造函数用传统语法**：需要 new 调用的函数，必须用 function 关键字
4. **逐渐习惯**：开始可能觉得箭头函数语法奇怪，用多了会发现它非常优雅

#### ES2017+ 异步编程：async/await 语法

在深入学习模块化之前，我们需要先了解 **async/await** 语法。这是ES2017引入的特性，用于简化异步操作的处理。

**为什么需要 async/await？**

现代前端开发中，我们经常需要处理异步操作，比如：
- 从服务器获取数据（fetch API）
- 读取文件（Node.js fs模块）
- 定时器操作

```javascript
// ❌ 传统方式：回调函数（容易陷入"回调地狱"）
fetch('/data/users.json')
  .then(response => response.json())
  .then(users => {
    console.log(users);
  })
  .catch(error => {
    console.error(error);
  });

// ✅ 现代方式：async/await（代码更像同步代码，更易读）
async function loadUsers() {
  try {
    const response = await fetch('/data/users.json');
    const users = await response.json();
    console.log(users);
  } catch (error) {
    console.error(error);
  }
}
```

**基本语法：**

```javascript
// 1. async 函数声明
async function fetchData() {
  // async 表示这是一个异步函数
  // 在函数内部可以使用 await 关键字
}

// 2. await 等待异步操作完成
async function example() {
  // await 会暂停函数执行，直到 Promise 解决
  const result = await someAsyncOperation();
  // 只有等待完成后，才会执行下一行
  console.log(result);
}

// 3. async 函数总是返回 Promise
async function getData() {
  return '数据';  // 实际返回的是 Promise<string>
}
```

**核心概念：**

```
async 关键字：
├── 声明一个函数是异步的
├── 允许在函数内使用 await 关键字
└── 自动将返回值包装成 Promise

await 关键字：
├── 只能在 async 函数内部使用
├── 暂停函数执行，等待 Promise 解决
└── 返回 Promise 的 resolved 值
```

**实际示例对比：**

```javascript
// 示例1：模拟网络请求
function simulateNetworkRequest() {
  return new Promise(resolve => {
    setTimeout(() => resolve('数据加载成功'), 1000);
  });
}

// 使用 async/await
async function loadData() {
  console.log('开始加载...');
  const result = await simulateNetworkRequest();  // 等待1秒
  console.log(result);  // 1秒后才执行
  return result;
}

loadData();

// 输出顺序：
// 开始加载...
// (等待1秒)
// 数据加载成功
```

**错误处理：**

```javascript
async function loadDataWithErrorHandling() {
  try {
    const response = await fetch('/data.json');
    const data = await response.json();
    return data;
  } catch (error) {
    // 捕获任何异步操作中的错误
    console.error('加载失败:', error);
    return null;
  }
}
```

**重要注意事项：**

```javascript
// ❌ 错误：不能在普通函数中使用 await
function badExample() {
  const result = await fetchData();  // 语法错误！
}

// ✅ 正确：必须在 async 函数中使用
async function goodExample() {
  const result = await fetchData();  // 正确
}

// ✅ 也可以使用箭头函数
const loadData = async () => {
  const result = await fetchData();
  return result;
};

// ✅ 顶层 await（ES2022+，模块中可用）
// 在 ES 模块的顶层可以直接使用 await
const data = await fetchData();  // 在 .js 文件（使用 ES Modules）的顶层
```

**在我们课程中的应用：**

接下来的模块化示例中，你会看到这样的代码：

```javascript
// 这些代码使用了 async/await 语法
export async loadData(resourceName) {
  const response = await fetch(`${API_BASE}/${resourceName}.json`);
  return await response.json();
}

export async getUsers() {
  return await loadData('users');
}
```

现在你知道：
- `async` 声明这些函数是异步的
- `await` 等待 fetch 操作完成
- 这些函数返回 Promise，调用时也需要使用 await

**学习建议：**

如果你对 async/await 还不熟悉，不必担心。在接下来的示例中：
1. 先理解模块化的概念（import/export）
2. async/await 可以暂时理解为"等待异步操作完成"的语法
3. 后续课程会深入讲解异步编程

### 2.2 ES Modules 详解

#### 基本语法

**导出（export）：**

```javascript
// utils/math.js

// 1. 命名导出（可以导出多个）
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
export const PI = 3.14159;

// 2. 也可以先定义再导出
const multiply = (a, b) => a * b;
const divide = (a, b) => a / b;
export { multiply, divide };

// 3. 导出时重命名
export { multiply as mul, divide as div };

// 4. 默认导出（一个文件只能有一个默认导出）
export default class Calculator {
  add(a, b) { return a + b; }
  subtract(a, b) { return a - b; }
}
```

**导入（import）：**

```javascript
// main.js

// 1. 导入命名导出
import { add, subtract, PI } from './utils/math.js';
console.log(add(5, 3));        // 8
console.log(subtract(5, 3));   // 2
console.log(PI);               // 3.14159

// 2. 导入时重命名
import { multiply as mul } from './utils/math.js';
console.log(mul(5, 3));        // 15

// 3. 导入全部到对象
import * as math from './utils/math.js';
console.log(math.add(5, 3));   // 8
console.log(math.PI);          // 3.14159

// 4. 导入默认导出
import Calculator from './utils/math.js';
const calc = new Calculator();
console.log(calc.add(5, 3));   // 8

// 5. 混合导入
import Calculator, { add } from './utils/math.js';

// 6. 只导入（不使用导出的值，用于执行副作用）
import './utils/logger.js';
```

**重要提示：**

```javascript
// ✅ 正确：导入时必须包含文件扩展名
import { add } from './utils/math.js';

// ❌ 错误：不要省略 .js
import { add } from './utils/math';

// ✅ 目录导入会自动查找 index.js
import { config } from './config';
// 实际加载的是 ./config/index.js
```

#### 模块化实战示例

创建一个完整的小项目：

```
project/
├── index.js          # 入口文件
├── utils/
│   ├── math.js       # 数学工具
│   ├── string.js     # 字符串工具
│   └── index.js      # 工具聚合导出
└── data/
    └── users.js      # 数据文件
```

```javascript
// utils/math.js
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
export const multiply = (a, b) => a * b;
export const divide = (a, b) => b !== 0 ? a / b : null;
```

```javascript
// utils/string.js
export const capitalize = (str) => {
  return str.charAt(0).toUpperCase() + str.slice(1);
};

export const truncate = (str, length) => {
  return str.length > length ? str.slice(0, length) + '...' : str;
};

export const generateId = () => {
  return Math.random().toString(36).substr(2, 9);
};
```

```javascript
// utils/index.js
// 聚合导出（barrel pattern）
export * from './math.js';
export * from './string.js';
```

```javascript
// data/users.js
export const users = [
  { id: 1, name: 'alice', age: 25 },
  { id: 2, name: 'bob', age: 30 },
  { id: 3, name: 'charlie', age: 35 },
];

export const getUserById = (id) => {
  return users.find(user => user.id === id);
};
```

```javascript
// index.js
import * as utils from './utils/index.js';
import { users, getUserById } from './data/users.js';

console.log('=== 数学工具测试 ===');
console.log('5 + 3 =', utils.add(5, 3));
console.log('5 - 3 =', utils.subtract(5, 3));
console.log('5 * 3 =', utils.multiply(5, 3));
console.log('5 / 3 =', utils.divide(5, 3));

console.log('\n=== 字符串工具测试 ===');
console.log(utils.capitalize('hello'));
console.log(utils.truncate('This is a long text', 10));
console.log('Generated ID:', utils.generateId());

console.log('\n=== 用户数据测试 ===');
console.log('所有用户:', users);
console.log('查找用户ID=2:', getUserById(2));
```

运行：

```bash
node index.js
```

#### 模块的异步加载特性

ES Modules是同步加载的，这与传统的`<script>`标签不同：

```javascript
// 模块A
console.log('模块A开始');

import { something } from './moduleB.js';
// 这里会等待 moduleB.js 加载完成

console.log('模块A结束');

// 输出顺序：
// 模块B开始
// 模块B结束
// 模块A开始
// 模块A结束
```

#### 动态导入（Dynamic Import）

ES2020引入了动态导入，可以在运行时按需加载模块：

```javascript
// 静态导入（编译时确定）
import { add } from './math.js';

// 动态导入（运行时加载）
const mathModule = await import('./math.js');
console.log(mathModule.add(5, 3));

// 使用场景：按需加载、条件加载
if (needsAdvancedFeature) {
  const advanced = await import('./advanced-features.js');
  advanced.doSomething();
}
```
