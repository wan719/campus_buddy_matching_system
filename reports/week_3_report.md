```gradle
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'

    implementation 'org.springframework.boot:spring-boot-starter-security'

    runtimeOnly 'io.jsonwebtoken:jjwt-impl:0.11.5'
```

### 3.2 典型问题深度分析

**问题描述**：
MySQL 连接时报错 `The server time zone value '�й���ʱ��' is unrecognized or represents more than one time zone`。项目启动时无法连接数据库，Spring Boot 启动失败。

**问题分析**：
MySQL 8.0 之后对时区要求更严格，服务器默认时区是系统时区（如 CST），但 JDBC 驱动无法正确识别该时区值，导致连接建立时抛出异常。

**解决过程**：
1. 尝试1：在 MySQL 命令行执行 `SET GLOBAL time_zone = '+8:00'`（结果：临时生效，但数据库重启后失效）
2. 尝试2：修改 MySQL 配置文件 my.cnf，添加 `default-time-zone='+8:00'`（结果：有效，但需要重启数据库服务且需要服务器权限）
3. 最终方案：在 JDBC URL 中添加参数 `serverTimezone=Asia/Shanghai`，这是最简便、无侵入的解决方案，无需修改数据库配置

**经验总结**：
- 数据库连接问题优先检查 URL 参数，很多问题可以通过连接字符串解决，避免修改数据库全局配置
- 遇到时区问题时，明确指定 serverTimezone 比依赖数据库默认值更可靠
- 在 application.yaml 中把数据库配置写清楚并添加注释，可以避免环境不一致导致的连接问题
